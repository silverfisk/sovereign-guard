ENROLLMENT PROCESS

1
Initial Setup:

Connect the ESP32 watchdog to the server via USB. The device generates a unique Ed25519 key pair on first boot.

2
Key Exchange:

The server daemon reads the watchdog's public key, and the admin provides their PGP public key for enrollment.

3
Trust Establishment:

The admin signs the watchdog's public key with their PGP key, establishing a trust relationship.

4
Pre-Sign Commands:

The admin creates and signs emergency response packages that the watchdog can execute autonomously.

5
Deployment:

The system becomes operational. The watchdog monitors the server and can execute pre-signed recovery commands.

PRE-SIGNED COMMAND EXAMPLES

Emergency Recovery Commands:

# Server reboot (signed by admin)
sudo systemctl reboot
# Service restart
sudo systemctl restart nginx
# Clear disk space
sudo /opt/cleanup-logs.sh

Validation & Execution:

# Verify PGP signature
gpg --verify command.sig command.sh
# Check timestamp validity
validate_timestamp()
# Execute if valid
bash command.sh

Time-Limited Commands: All pre-signed packages include expiration timestamps for enhanced security.

SECURITY PRINCIPLES

AIR-GAPPED CHANNEL

The watchdog communicates with the server via a direct USB link, isolating it from network-based attacks. This secure channel can be used for remote access with minimal exposure.

KEY PROTECTION

Hardware-protected key storage with secure boot. Private keys never leave the device. Supports key rotation and revocation.

SILICON-LEVEL PROTECTION

ESP32 Secure Boot & Flash Encryption create a hardware root of trust. Even physical access can't compromise your sovereignty.

IMMUTABLE TRUTH

Cryptographically signed audit trails. Every action is recorded, verified, and tamper-proof. Truth through mathematics.
